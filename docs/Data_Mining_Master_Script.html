Workflow chung của một bài Data Mining

Trong đa số bài, nên nghĩ theo pipeline sau:

  1. Importer / vérifier les données
    Kiểm tra kiểu dữ liệu, số dòng cột, missing values, biến định lượng hay định tính.

  2. Exploration univariée et bivariée
    Xem phân phối, ngoại lệ, mức độ phân tán, tương quan, bảng chéo.

  3. Prétraitement
    Centering, scaling, chọn biến, loại biến gần hằng, mã hóa biến định tính nếu cần.

  4. Méthode principale

    • ACP nếu biến định lượng, mục tiêu mô tả / giảm chiều.
    • AFC/AFCM nếu bảng chéo / biến định tính.
    • CAH / k-means nếu phân cụm.
    • LDA / kNN / CART nếu phân loại có giám sát.
    • PLS nếu hồi quy với đa cộng tuyến hay nhiều biến giải thích.

  5. Validation / choix du modèle
    Không được ngây thơ nhìn mỗi training accuracy. Phải xem CV, PRESS, xerror, stabilité.

  6. Interprétation substantielle
    Cuối cùng phải trả lời được: trục nào mang ý nghĩa gì, nhóm nào tách nhau ra sao, mô hình nào đáng tin nhất.

# Helper cho panel pairs
panel.hist <- function(x, ...) {
  usr <- par("usr"); on.exit(par(usr))
  par(usr = c(usr[1:2], 0, 1.5))
  h <- hist(x, plot = FALSE)
  breaks <- h$breaks; nB <- length(breaks)
  y <- h$counts; y <- y / max(y)
  rect(breaks[-nB], 0, breaks[-1], y, col = "lightgray", ...)
}

panel.cor <- function(x, y, digits = 2, prefix = "r=", cex.cor, ...) {
  usr <- par("usr"); on.exit(par(usr))
  par(usr = c(0, 1, 0, 1))
  r <- cor(x, y, use = "pairwise.complete.obs")
  txt <- format(c(r, 0.123456789), digits = digits)[1]
  txt <- paste0(prefix, txt)
  if (missing(cex.cor)) cex.cor <- 1.2
  text(0.5, 0.5, txt, cex = cex.cor)
}

# CV lặp lại cho kNN
knn_cv_mean <- function(X, y, k = 3, nbcv = 4, nbrun = 30, seed = 123) {
  set.seed(seed)
  X <- as.matrix(X)
  y <- as.factor(y)
  acc <- numeric(nbrun)
  n <- nrow(X)

  for (run in seq_len(nbrun)) {
    folds <- sample(rep(1:nbcv, length.out = n))
    fold_acc <- numeric(nbcv)

    for (f in seq_len(nbcv)) {
      test_id <- which(folds == f)
      train_id <- setdiff(seq_len(n), test_id)
      pred <- knn(train = X[train_id, , drop = FALSE],
                  test  = X[test_id, , drop = FALSE],
                  cl    = y[train_id],
                  k     = k)
      fold_acc[f] <- mean(pred == y[test_id])
    }
    acc[run] <- mean(fold_acc)
  }
  mean(acc)
}

# CV lặp lại cho LDA
lda_cv_mean <- function(X, y, nbcv = 4, nbrun = 30, seed = 123) {
  set.seed(seed)
  X <- as.matrix(X)
  y <- as.factor(y)
  acc <- numeric(nbrun)
  n <- nrow(X)

  for (run in seq_len(nbrun)) {
    folds <- sample(rep(1:nbcv, length.out = n))
    fold_acc <- numeric(nbcv)

    for (f in seq_len(nbcv)) {
      test_id <- which(folds == f)
      train_id <- setdiff(seq_len(n), test_id)
      fit <- lda(x = X[train_id, , drop = FALSE], grouping = y[train_id])
      pred <- predict(fit, newdata = X[test_id, , drop = FALSE])$class
      fold_acc[f] <- mean(pred == y[test_id])
    }
    acc[run] <- mean(fold_acc)
  }
  mean(acc)
}

8. Mẫu khung interpretation cho từng phương pháp

8.1. Exploration

Les variables présentent des dispersions différentes, ce qui justifie un centrage-réduction avant les méthodes fondées sur les distances ou les variances. Les corrélations élevées entre plusieurs variables suggèrent une redondance d’information et plaident en faveur d’une ACP.

8.2. ACP

Les deux premiers axes résument xx% de l’inertie totale. L’axe 1 correspond principalement à […], tandis que l’axe 2 oppose […]. Les individus situés à proximité présentent des profils semblables. Les variables proches du cercle sont bien représentées sur le plan factoriel.

8.3. AFC / AFCM

L’analyse met en évidence une structure d’association entre les profils de lignes et de colonnes. Les modalités / individus proches dans le plan factoriel présentent des profils similaires relativement à la structure moyenne du tableau.

8.4. CAH / k-means

La classification met en évidence x groupes relativement homogènes. La méthode de Ward donne une partition plus équilibrée et plus interprétable que les autres méthodes d’agrégation. Les résultats sont cohérents avec la projection ACP.

8.5. LDA

LDA fournit une bonne séparation entre les groupes sur les premiers axes discriminants. Le taux de bien classés sur l’échantillon d’apprentissage est de xx%, mais le taux de validation croisée est plus pertinent pour juger la capacité de généralisation.

8.6. kNN

Le choix du nombre de voisins a été optimisé par validation croisée. Les meilleurs résultats sont obtenus pour k = x. Un k trop faible rend le modèle instable, alors qu’un k trop élevé lisse excessivement la frontière de décision.

8.7. CART

L’arbre complet est trop complexe et susceptible de surapprentissage. Le choix d’un arbre élagué repose sur l’erreur de validation croisée (xerror) afin d’obtenir un compromis entre performance et interprétabilité.

8.8. PLS

Le nombre de composantes a été choisi à partir du PRESS moyen et du R² en validation croisée. Bien que les performances d’apprentissage augmentent avec le nombre de composantes, le gain devient marginal au-delà de x composantes, ce qui justifie un modèle plus parcimonieux.

9. Pièges fréquents à éviter

  1. Confondre training accuracy và validation accuracy.
    Đây là lỗi rất hay gặp và rất dở.

  2. Quên scale dữ liệu trước kNN / CAH / k-means / ACP sur corrélations.

  3. Diễn giải trục mà không nhìn loadings / corrélations.
    Chỉ nhìn plot rồi bịa nghĩa là con đường ngắn nhất tới nhận xét vô giá trị.

  4. Chọn số composantes chỉ theo training fit trong PLS.

  5. Không loại biến phụ thuộc tuyến tính / quasi-constantes trước LDA.

  6. Tin một chỉ số duy nhất để chọn số cụm.

10. Kết luận học nhanh

Nếu cần học nhanh để đi thi hoặc viết rapport, hãy nhớ logic ngắn gọn này:

  • ACP: dữ liệu định lượng, nhiều biến tương quan, mục tiêu mô tả cấu trúc và giảm chiều.
  • AFC/AFCM: dữ liệu định tính / bảng chéo.
  • CAH / k-means: phân nhóm không giám sát.
  • LDA / kNN / CART: phân loại có giám sát.
  • PLS: hồi quy khi X nhiều và tương quan mạnh.

Thứ tự làm gần như luôn là:
exploration -> prétraitement -> méthode -> validation -> interpretation.
