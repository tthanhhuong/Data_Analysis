Fil directeur du chapitre

Un test statistique repose toujours sur le même enchaînement :

  1. formuler deux hypothèses : \(H_0\) et \(H_1\) ;
  2. choisir une variable de test (VT) ;
  3. connaître la loi de la VT sous \(H_0\) ;
  4. placer la valeur observée de la VT dans cette loi ;
  5. obtenir une p value ;
  6. conclure en assumant les risques \(\alpha\) et \(\beta\).

Mémo R indispensable : la logique des fonctions d/p/q/r

Pour presque toutes les lois sous R, on retrouve la même logique :

  • d... : densité ou masse de probabilité en un point ;
  • p... : probabilité cumulée ;
  • q... : quantile ;
  • r... : tirage aléatoire.

Exemples :

  • dbinom() = probabilité exacte pour une loi binomiale ;
  • pnorm() = probabilité cumulée sous une loi normale ;
  • qchisq() = seuil critique d’une loi du \(\chi^2\) ;
  • rt() = simulation aléatoire sous loi de Student.

1. Les deux hypothèses statistiques

Rappel conceptuel

  • \(H_0\) : hypothèse nulle, celle que le test cherche à mettre à l’épreuve.
  • \(H_1\) : hypothèse alternative, celle qu’on retient si les données sont trop improbables sous \(H_0\).

Point important : on ne “prouve” presque jamais \(H_1\). On rejette ou non \(H_0\) selon la compatibilité entre les données et la loi théorique de la VT sous \(H_0\).

À retenir

  • La p value ne mesure pas la probabilité que \(H_0\) soit vraie.
  • Elle mesure la probabilité d’observer une valeur de VT au moins aussi extrême que celle obtenue, si \(H_0\) est vraie.

2. La variable de test (VT)

Définition

La VT est une fonction calculée à partir de l’échantillon.
Exemples classiques :

  • une moyenne standardisée ;
  • une statistique \(t\) ;
  • un \(\chi^2\) ;
  • une somme de rangs de Wilcoxon ;
  • une statistique \(U\) de Mann-Whitney.

Idée centrale

La variable mesurée et la variable de test ne suivent pas forcément la même loi.

  • Une variable biologique peut être asymétrique.
  • Sa VT, après standardisation, peut suivre une loi de Student ou normale sous certaines conditions.

C’est un point que beaucoup de gens ratent, ce qui est assez prévisible dès qu’on confond données brutes et statistique de test.

3. Distributions de probabilité discrètes

3.1 Loi binomiale

Formule manuelle puis via R

p <- 0.2
n <- 4
k <- 2

choose(n, k) * p^k * (1 - p)^(n - k)
#> [1] 0.1536
dbinom(k, n, p)
#> [1] 0.1536

Interprétation.
On calcule ici \(P(X = 2)\) avec \(X \sim \mathcal{B}(n=4, p=0.2)\).
Les deux lignes doivent donner la même valeur :

  • la première applique directement la formule théorique ;
  • la seconde utilise la fonction R optimisée.

À retenir. dbinom(k, n, p) donne une probabilité exacte, pas une probabilité cumulée.

Représentation graphique

p <- 0.2
n <- 4
k <- 0:4
plot(
  k, dbinom(k, n, p),
  pch = 16, cex = 2,
  xlim = range(0, 5), ylim = range(0, 0.5),
  xlab = "Nombre de boules noires k",
  ylab = "Probabilité f(k)",
  cex.lab = 1.2, cex.axis = 1.1,
  bty = "l"
)

Interprétation.
La loi binomiale est discrète : on observe des masses de probabilité sur des valeurs entières de \(k\).
Le graphe rappelle qu’il n’y a rien entre 0, 1, 2, 3 et 4. Les courbes lisses sont jolies, mais les variables discrètes, elles, n’ont pas signé pour ça.

Probabilité à droite de la distribution

p <- 0.2
n <- 4

dbinom(2, n, p) + dbinom(3, n, p) + dbinom(4, n, p)
#> [1] 0.1808
1 - dbinom(0, n, p) - dbinom(1, n, p)
#> [1] 0.1808
pbinom(1, n, p, lower.tail = FALSE)
#> [1] 0.1808

Interprétation.
Ces trois instructions calculent la même quantité :

\[ P(X \ge 2) \]

lower.tail = FALSE signifie qu’on travaille dans la queue droite de la distribution.

Quantiles binomiaux

p <- 0.2
n <- 4

qbinom(0.1808, n, p, lower.tail = FALSE)
#> [1] 1
qbinom(0.15, n, p, lower.tail = FALSE)
#> [1] 2
qbinom(0.5, n, p)
#> [1] 1

Interprétation.
qbinom() cherche le seuil entier associé à une probabilité cumulée donnée.
C’est utile pour définir des valeurs critiques dans les tests exacts.

Influence de n

p <- 0.2
k <- 0:20

plot(
  k, dbinom(k, 4, p), type = "l",
  xlab = "Nombre de boules noires k",
  ylab = "Probabilité f(k)",
  cex.lab = 1.2, cex.axis = 1.1
)
lines(k, dbinom(k, 20, p), lwd = 3)
lines(k, dbinom(k, 50, p), lty = "39", lwd = 2)
legend(
  "topright",
  legend = c("n=4", "n=20", "n=50"),
  lty = c(1, 1, 2), lwd = c(1, 3, 2), bty = "n"
)

Interprétation.
Quand n augmente, la distribution se décale et se concentre autour de \(np\).
On commence aussi à voir apparaître un comportement plus “gaussien” par approximation.

3.2 Loi multinomiale

Probabilité d’une configuration particulière

k1 <- 2; k2 <- 7; k3 <- 1
p1 <- 0.1; p2 <- 0.4; p3 <- 0.5

factorial(k1 + k2 + k3) /
  (factorial(k1) * factorial(k2) * factorial(k3)) *
  p1^k1 * p2^k2 * p3^k3
#> [1] 0.00294912
dmultinom(c(k1, k2, k3), prob = c(p1, p2, p3))
#> [1] 0.00294912

Interprétation.
La loi multinomiale généralise la binomiale à plus de deux catégories.
On calcule ici la probabilité d’obtenir exactement \((2,7,1)\) observations dans trois classes.

Surface des probabilités multinomiales

library(scatterplot3d)

p1 <- 0.1; p2 <- 0.4; p3 <- 1 - p1 - p2
n <- 10
x <- NULL; y <- NULL; z <- NULL

for (k2 in 0:n) {
  for (k1 in 0:n) {
    x <- c(x, k1)
    y <- c(y, k2)
    if ((n - k1 - k2) >= 0) {
      z <- c(z, dmultinom(c(k1, k2, n - k1 - k2), prob = c(p1, p2, p3)))
    } else {
      z <- c(z, 0)
    }
  }
}

scatterplot3d(
  x, y, z,
  type = "h", pch = 16,
  xlab = "Nombre de boules noires k1",
  ylab = "",
  zlab = "Probabilité",
  highlight.3d = TRUE,
  box = FALSE
)
par(xpd = NA)
text(x = 7.5, y = 0.5, labels = "Nombre de boules rouges k2", srt = 38)

Note pédagogique.
Chunk laissé en eval=FALSE par défaut car il demande le package scatterplot3d.
L’intérêt conceptuel est de voir que la probabilité est répartie sur un simplexe discret, pas sur une ligne comme dans la binomiale.

3.3 Loi de Pascal et loi binomiale négative

p <- 0.2
n <- 4
k <- 2
r <- 2

choose(n - 1, k - 1) * p^k * (1 - p)^(n - k)
#> [1] 0.0768
dnbinom(n - k, k, p)
#> [1] 0.0768
dnbinom(r, k, p)
#> [1] 0.0768

Interprétation.
La loi de Pascal ou binomiale négative s’intéresse au nombre d’échecs avant un certain nombre de succès.
Selon l’écriture choisie :

  • n-k = nombre d’échecs ;
  • k = nombre de succès attendus.

Queue droite et visualisation

p <- 0.2
n <- 1:80
k <- 2

plot(
  n, dnbinom(n - k, k, p), type = "l",
  xlab = "Nombre de tirages n (loi de Pascal)",
  ylab = "Probabilité f(n)",
  cex.lab = 1.2, cex.axis = 1.1
)
lines(n, dnbinom(n - k, 1, 0.5), lwd = 3)
lines(n, dnbinom(n - 2, 2, 0.02), lty = "39", lwd = 2)
lines(n, dnbinom(n - 15, 15, 0.2), lty = "83", lwd = 2)
legend(
  "topright",
  legend = c("k=2, p=0.2", "k=1, p=0.5", "k=2, p=0.02", "k=15, p=0.2"),
  lty = c(1, 1, 2, 3), lwd = c(1, 3, 2, 2), bty = "n"
)

Interprétation.
Plus le succès est rare, plus la distribution s’étale vers la droite.
Autrement dit, si \(p\) est faible, il faut souvent beaucoup de tirages pour accumuler les succès attendus.

3.5 Loi hypergéométrique

k <- 2
Np <- 10
N <- 50
n <- 4

choose(Np, k) * choose(N - Np, n - k) / choose(N, n)
#> [1] 0.1524099
dhyper(k, Np, N - Np, n)
#> [1] 0.1524099

Interprétation.
La loi hypergéométrique s’applique aux tirages sans remise.
C’est la vraie loi derrière les problèmes de contingence et certains tests exacts.

Interprétation.
Plus l’échantillon prélevé est grand relativement à la population, plus l’effet du sans remise devient important.

3.6 Loi de Poisson

k <- 2
p <- 0.02
n <- 50

(n * p)^k * exp(-n * p) / factorial(k)
#> [1] 0.1839397
dpois(k, n * p)
#> [1] 0.1839397

Interprétation.
La loi de Poisson approxime une binomiale quand :

  • n est grand ;
  • p est petit ;
  • np reste modéré.

Elle est centrale pour modéliser des événements rares.

Visualisation

k <- 0:20
plot(
  k, dpois(k, 50 * 0.02), type = "l",
  xlab = "Nombre de boules noires k",
  ylab = "Probabilité f(k)",
  cex.lab = 1.2, cex.axis = 1.1
)
lines(k, dpois(k, 50 * (1 / 200)), lwd = 3)
lines(k, dpois(k, 200 * 0.02), lty = "39", lwd = 2)
lines(k, dpois(k, 500 * 0.02), lty = "83", lwd = 2)
legend(
  "topright",
  legend = c("np=1", "np=0.25", "np=4", "np=10"),
  lty = c(1, 1, 2, 3), lwd = c(1, 3, 2, 2), bty = "n"
)

Interprétation.
Dans une loi de Poisson, la moyenne et la variance sont toutes deux égales à \(\lambda = np\).
C’est utile, mais aussi dangereux en pratique : si la variance dépasse fortement la moyenne, il faut penser à la surdispersion.

4. Distributions de probabilité continues

4.1 Loi normale

Densité ponctuelle et probabilité cumulée

x <- 160
mu <- 165
sigma <- 7

1 / (sqrt(2 * pi) * sigma) * exp(-((x - mu)^2 / (2 * sigma^2)))
#> [1] 0.04415934
dnorm(x, mu, sigma)
#> [1] 0.04415934
pnorm(x, mu, sigma, lower.tail = FALSE)
#> [1] 0.7624747
pnorm(x, mu, sigma)
#> [1] 0.2375253

Interprétation.
dnorm() renvoie une densité, pas une probabilité ponctuelle au sens discret.
Pour une variable continue, \(P(X = 160)\) est théoriquement nulle. Ce que mesure dnorm(160, ...), c’est la hauteur de la courbe en 160.

Intervalle de référence autour de 95%

mu <- 165
sigma <- 7

pnorm(mu + 1.96 * sigma, mu, sigma) - pnorm(mu - 1.96 * sigma, mu, sigma)
#> [1] 0.9500042
mu + 1.96 * sigma
#> [1] 178.72

Interprétation.
On retrouve ici l’idée classique que, sous normalité,

\[ P(\mu - 1.96\sigma \le X \le \mu + 1.96\sigma) \approx 0.95 \]

Quantiles normaux

F <- 0.5
mu <- 165
sigma <- 7
qnorm(F, mu, sigma, lower.tail = TRUE)
#> [1] 165
F <- 0.025
qnorm(F, mu, sigma, lower.tail = FALSE)
#> [1] 178.7197

Interprétation.
qnorm() est la fonction qui permet d’obtenir les seuils critiques d’une loi normale.

Comparaison de normales

x <- seq(-10, 10, 0.01)
plot(
  x, dnorm(x, 0, 1), type = "l",
  xlab = "Variable continue x",
  ylab = "Densité f(x)",
  cex.lab = 1.2, cex.axis = 1.1
)
lines(x, dnorm(x, 4, 1), lwd = 3)
lines(x, dnorm(x, 0, 4), lty = "39", lwd = 2)
legend(
  "topright",
  legend = c("mu=0, sigma=1", "mu=4, sigma=1", "mu=0, sigma=4"),
  lty = c(1, 1, 2), lwd = c(1, 3, 2), bty = "n"
)

Interprétation.
- Modifier mu décale la courbe. - Modifier sigma change l’étalement.

4.5 Loi du \(\chi^2\)

x <- 2
ddl <- 4

1 / (2^(ddl / 2) * gamma(ddl / 2)) * x^(ddl / 2 - 1) * exp(-x / 2)
#> [1] 0.1839397
dchisq(x, ddl)
#> [1] 0.1839397
pchisq(x, ddl)
#> [1] 0.2642411
pchisq(x, ddl, lower.tail = FALSE)
#> [1] 0.7357589
qchisq(0.025, ddl, lower.tail = FALSE)
#> [1] 11.14329

Interprétation.
La loi du \(\chi^2\) apparaît dans :

  • les tests d’ajustement ;
  • les tableaux de contingence ;
  • les estimations de variance.

Elle est toujours positive et asymétrique à droite, surtout pour petits ddl.

4.7 Loi de Student

La loi de Student ressemble à une normale, mais avec des queues plus épaisses quand les ddl sont faibles.

5.3 Construction combinatoire exacte de la distribution de W/U

Cette section reprend le code long du livre. Elle est très utile pour comprendre comment naît la distribution exacte, mais elle devient vite coûteuse en mémoire.

n1 <- 2
n2 <- 2
n <- n1 + n2

time.ini <- as.numeric(Sys.time())
nb.seq <- choose(n, n1)

if (nb.seq < 2^31 - 1) {
  cat("Nombre de combinaisons : ", nb.seq, "\n\n")
  comb <- matrix("O", nrow = nb.seq, ncol = n)
} else {
  cat("Nombre de séquences possibles", nb.seq,
      "au-dessus de la limite de dimension d'une matrice dans R\n\n")
}

rang <- 1:n
pos.seq <- 1:length(rang)
pos.seq.diff <- pos.seq[!pos.seq %in% rang]
rang.diff <- rang[!rang %in% pos.seq]

library(gtools)

if (n1 <= n2) {
  n.tempo <- n1
  cat("Dans comb, O représente la classe 2 et A la classe 1\n\n")
} else {
  n.tempo <- n2
  cat("Dans comb, O représente la classe 1 et A la classe 2\n\n")
}

pos.col.A <- combinations(n, n.tempo, pos.seq, set = FALSE)
comb[matrix(
  ncol = 2,
  c(rep(1:nb.seq, each = ncol(pos.col.A)), as.vector(t(pos.col.A)))
)] <- "A"

comb <- as.data.frame(comb)

if (length(pos.seq.diff) != 0) {
  for (i in 1:length(rang.diff)) {
    pos.col.A[pos.col.A == pos.seq.diff[i]] <- rang.diff[i]
  }
}

serie.W <- apply(pos.col.A, 1, sum)
serie.U <- serie.W - 0.5 * n1 * (n1 + 1)
comb.W.U <- cbind(comb, serie.W, serie.U)
if (nb.seq < 100) comb.W.U

W <- sort(unique(serie.W))
prob.W <- table(serie.W) / length(serie.W)
prob.W

plot(W, as.vector(prob.W), type = "b", pch = 16,
     xlab = "Valeur de W", ylab = "Probabilité f(W)")

library(lubridate)
time.fin <- as.numeric(Sys.time())
cat("Temps de calcul total : ",
    as.character(seconds_to_period(round(time.fin - time.ini))), "\n\n")

Interprétation.
Ce chunk montre le mécanisme exact du test :

  1. on énumère toutes les affectations possibles des rangs entre deux groupes ;
  2. on calcule W puis U pour chaque configuration ;
  3. on déduit la distribution exacte sous \(H_0\).

Message pédagogique.
C’est probablement la meilleure section pour comprendre qu’une loi exacte n’est pas un objet mystique tombé du ciel. C’est juste de la combinatoire, plus un peu de RAM sacrifiée au dieu du calcul.

5.5 Construction combinatoire exacte de V

n <- 3
time.ini <- as.numeric(Sys.time())
seed <- vector("list", n)
for (i in 1:n) seed[[i]] <- c("A", "O")

library(gtools)
nb.seq <- 2^n

if (nb.seq < 2^31 - 1) {
  cat("Nombre de séquences :", nb.seq, "\n\n")
  comb <- expand.grid(seed)
  comb.t <- t(comb)
} else {
  cat("Nombre de séquences possibles au-dessus de la limite\n\n")
}

serie.V <- NULL
for (i in 1:ncol(comb.t)) {
  if (any(comb.t[, i] == "A")) {
    serie.V <- c(serie.V, tapply(rang, comb.t[, i], sum)[1])
  } else {
    serie.V <- c(serie.V, 0)
  }
}

if (ncol(comb.t) < 100) {
  comb.V <- cbind(as.data.frame(t(comb.t)), serie.V)
  comb.V
}

V <- sort(unique(serie.V))
prob.V <- table(serie.V) / length(serie.V)
prob.V
plot(V, as.vector(prob.V), type = "b", pch = 16,
     xlab = "Valeur de V", ylab = "Probabilité f(V)")

library(lubridate)
time.fin <- as.numeric(Sys.time())
cat("Temps de calcul total :",
    as.character(seconds_to_period(round(time.fin - time.ini))), "\n\n")

Interprétation.
Même logique que précédemment : on énumère toutes les combinaisons de signes possibles, puis on calcule la somme des rangs associés.

Simulation de la fluctuation d’une VT t due à l’échantillonnage

8.3 Corrections multiples

Bonferroni

0.05 / (1:10)
#>  [1] 0.050000000 0.025000000 0.016666667 0.012500000 0.010000000 0.008333333
#>  [7] 0.007142857 0.006250000 0.005555556 0.005000000
p.adjust(c(0.03, 0.8, 0.001, 0.015), method = "bonferroni")
#> [1] 0.120 1.000 0.004 0.060

Interprétation.
Bonferroni est simple et robuste, mais souvent très conservateur.

Holm

p.adjust(c(0.8, 0.03, 0.015, 0.001), method = "holm")
#> [1] 0.800 0.060 0.045 0.004

Interprétation.
Holm contrôle aussi l’erreur familiale, mais est moins brutal que Bonferroni.

Benjamini-Hochberg

p.adjust(c(0.8, 0.03, 0.015, 0.001), method = "BH")
#> [1] 0.800 0.040 0.030 0.004

Interprétation.
BH contrôle le False Discovery Rate.
C’est souvent plus adapté quand on fait beaucoup de tests et qu’on accepte qu’une fraction des découvertes soit fausse.

#>              sd = 7

#>     alternative = one.sided

Interprétation.
Les deux approches doivent donner des résultats cohérents :

  • approche analytique avec ncp ;
  • approche directement encapsulée dans power.t.test().

12.2 Approximation de probabilité ponctuelle en variable continue

mean <- 43
sigma <- 2.3

pnorm(43.5, mean, sigma) - pnorm(42.5, mean, sigma)
#> [1] 0.1720966
pnorm(43.05, mean, sigma) - pnorm(42.95, mean, sigma)
#> [1] 0.01734395
(43.5 - 42.5) * dnorm(43, mean, sigma)
#> [1] 0.1734532

Interprétation.
Pour une variable continue, on ne calcule pas une probabilité ponctuelle, mais une probabilité sur un intervalle.
La densité sert d’approximation locale quand l’intervalle est petit.

for (i in 1:500) {
  obs <- rmultinom(1, n, prob = proba.theo)

}

hist(X2.obs, freq = FALSE,
     main = expression(paste("Fluctuation de la VT ", chi^2)),

     ylab = "Densité de la distribution")

lines(VT.X2, dchisq(VT.X2, ddl))

       lty = c(NA, 1), lwd = c(NA, 2), pch = c(22, NA), pt.cex = c(2.5, NA),
       pt.bg = c("white", NA), bty = "n")

Interprétation.
Très bon chunk pour comprendre que la statistique \(\chi^2\) obtenue en pratique fluctue autour de la loi théorique attendue sous \(H_0\).

13. Synthèse ultra-courte à mémoriser

Ce qu’il faut savoir faire en pratique

  1. Identifier le bon type de loi : discrète ou continue.
  2. Savoir lire d, p, q, r.
  3. Distinguer :
    • variable mesurée ;
    • variable de test ;
    • loi de la VT sous \(H_0\).
  4. Comprendre que la p value est une aire de queue.
  5. Savoir que :
    • \(\alpha\) = faux positif ;
    • \(\beta\) = faux négatif ;
    • puissance = \(1 - \beta\).
  6. Retenir les effets principaux sur la puissance :
    • n augmente -> puissance augmente ;
    • alpha diminue -> puissance diminue ;
    • delta augmente -> puissance augmente ;
    • variabilité augmente -> puissance diminue.

Ce qu’il faut éviter

  • confondre dnorm() avec une probabilité ponctuelle ;
  • oublier lower.tail = FALSE quand on veut la queue droite ;
  • utiliser unilatéral après avoir vu les données ;
  • traiter la p value comme un score de vérité ;
  • ignorer les comparaisons multiples.
